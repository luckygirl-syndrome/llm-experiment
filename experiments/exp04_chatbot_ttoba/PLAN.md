## 1. 시나리오 설계 원칙

### 1-1. 두 축: Impulse × Match 4분면

또바의 핵심 역할은 점수 조합에 따라 다른 브레이크/격려 판단을 내리는 것이다.
4분면을 모두 커버해야 응답 패턴의 차이를 검증할 수 있다.

| 분면 | Impulse | Match | 또바 예상 방향 |
|------|---------|-------|----------------|
| Q1 | 고 (70+) | 고 (65+) | 잘 맞지만 충동 요인 짚기 |
| Q2 | 고 (70+) | 저 (35−) | 강한 브레이크 |
| Q3 | 저 (35−) | 고 (65+) | 조건부 격려 |
| Q4 | 저 (35−) | 저 (35−) | 명확한 비추 |

### 1-2. SBTI 유형 선택 원칙

- **10개 시나리오, 10가지 유형 모두 다름** (중복 없음)
  - 같은 유형이 반복되면 "응답이 유형 때문인지 점수 때문인지" 구분 불가
- 4분면 × 2유형 = 8개 (비교 쌍): 같은 상품 + 다른 유형 → 유형별 차이 검증
- 긴장관계 × 2 = 2개 (S09 IT, S10 UM): 전용 시나리오로 분리

**사용 유형 (10종):**

| 시나리오 | 유형 | 분류 |
|---------|------|------|
| S01 | DUTE | Q1 비교-A |
| S02 | NIMO | Q1 비교-B |
| S03 | DUME | Q2 비교-A |
| S04 | NUTO | Q2 비교-B |
| S05 | DUTO | Q3 비교-A |
| S06 | NIME | Q3 비교-B |
| S07 | NUMO | Q4 비교-A |

| S09 | DITE | IT 긴장관계 |
| S10 | DUMO | UM 긴장관계 |

### 1-3. 상품 설계 기준

Impulse 고 조건:
- discount_rate 30~45%
- review_count 1,000+, rating 4.5+
- like_count 3,000+
- trend_hype/bundle/confidence = 1

Impulse 저 조건:
- discount_rate 0~10%
- review_count 30~150
- like_count 100 이하
- 마케팅 키워드 없음

Match 고 조건:
- style_similarity 25~35 (유저 스타일 태그와 상품 딱 맞음)
- interest "2주 이상 고민" → interest_persistence = 20
- discovery "직접 검색" → discovery_stability = 20
- price_feeling "저렴" or "괜찮음" → price_reasonable 18~25

Match 저 조건:
- style_similarity 0~8
- interest "오늘 처음" → interest_persistence = 4
- discovery "SNS 수동 노출" → discovery_stability = 6
- price_feeling "비싸요" → price_reasonable 3

| style_context | 힙·스트릿 취향 유저와 섹시글램 미니스커트의 스타일이 다릅니다. |
| **예상 Impulse** | **22~34** (D형 base +5, 그러나 I형 review_count_m 0.7, 할인 없음) |
| **예상 Match** | **10~18** |

> **비교 포인트 S07 vs S08:**
> - NUMO: "오늘 처음 봤고, 리뷰 평점도 낮고, 네 스타일이랑도 멀어. 뭐가 끌렸어?"
> - DIME: "D형이 오늘 처음 본 거니까 아직 충동도 아직 안 붙었잖아. 이거 네 기준에서 뭐가 맞아?"

---

### S09 — IT 긴장관계 / DITE (고집 센 고양이)

> DITE = D(충동) + I(자기판단) + T(유행참고) + E(거래이득)
> IT 긴장: 유행 안에서 후보를 찾지만, 최종 선택은 자기 기준으로 깐깐하게 검토

**목적:** 유행 아이템이라 T형으로 관심이 생겼는데, I형이라 자기 기준 검토 중 → 애매한 케이스. 또바가 "유행 때문에 끌린 건지, 진짜 네 기준에서 맞는 건지" 두 가지를 함께 짚는가?

```
상품명:    [랭킹1위/이번시즌 필템] 와이드 데님 팬츠
플랫폼:    무신사
원가:      72,000원
할인가:    57,600원 (20% 할인)
리뷰:      980개 / 평점 4.5
찜:        2,800개
마케팅:    trend_hype=["랭킹1위", "이번시즌 필템"], bundle=[], confidence=[]
상품 설명: 와이드 데님 팬츠. 이번 시즌 트렌드 아이템. 캐주얼·스트릿 무드.
```

| 유저 파라미터 | 값 |
|-------------|-----|
| 유형 | DITE |
| 나이 | 25~29 |
| 선호 스타일 | 캐주얼, 스트릿 |
| 구매 후회 | 1~2번 / 충동적으로 샀는데 안 입게 됨 |
| 발견 경로 | 쇼핑 앱 랭킹/유저 추천 |
| 관심 기간 | 1주일 정도 됐어요 |
| 가격 체감 | 이 정도면 괜찮아요 |
| 접촉 이유 | 오래 고민했는데 결정이 안 나서요 |
| style_similarity | 18 (스타일 맞는 편이지만 완벽하지 않음 → 애매한 케이스) |
| style_context | 캐주얼·스트릿 선호 유저에게 어느 정도 맞지만, 기존 옷장과의 조합을 확인할 필요 있습니다. |
| **예상 Impulse** | **55~65** (D형이지만 I형 review_count_m 0.7, 20% 할인은 가벼운 자극) |
| **예상 Match** | **40~52** (style_similarity 중간, 랭킹 발견은 discovery_stability 중간) |

**IT 긴장관계 체크:**
- T형 → "랭킹 1위 / 이번 시즌 필템" 문구에 관심이 시작됨
- I형 → "근데 랭킹이 높다고 내 기준에 맞는 건 아니잖아"
- 또바가 이 두 축의 긴장을 짚어야 함: "유행이라서 눈에 들어온 건지, 네 기준에서 직접 따져봤을 때 맞는 건지 구분해봐"

---

### S10 — UM 긴장관계 / DUMO (깐깐한 고슴도치)

> DUMO = D(충동) + U(타인검증) + M(차별화) + O(물건가치)
> UM 긴장: 타인 검증을 원하지만 리뷰가 너무 많으면 오히려 흥미 감소

**목적:** 소수 검증 구간(리뷰 ~95개)의 한정 느낌 아이템. U형으로 검증 확인 원하는데 M형으로 "이게 너무 대중화되면 싫어질 텐데". 이미 결정한 유저 + 또바가 조용히 근거 짚어주기.

```
상품명:    [한정수량/무신사 단독] 빈티지워시 데님 재킷
플랫폼:    무신사
원가:      89,000원
할인가:    71,200원 (20% 할인)
리뷰:      95개 / 평점 4.8
찜:        780개
마케팅:    trend_hype=[], bundle=[], confidence=["한정수량", "무신사 단독"]
상품 설명: 빈티지워시 가공 데님 재킷. 소량 입고, 단독 판매.
```

| 유저 파라미터 | 값 |
|-------------|-----|
| 유형 | DUMO |
| 나이 | 20~24 |
| 선호 스타일 | 힙, 빈티지 |
| 구매 후회 | 1~2번 / 충동적으로 샀는데 안 입게 됨 |
| 발견 경로 | 카테고리 검색 |
| 관심 기간 | 1주일 정도 됐어요 |
| 가격 체감 | 좀 비싸긴 한데 못 살 정도는 아니에요 |
| 접촉 이유 | 이미 마음은 정했는데 마지막으로 한 번만 봐줘요 |
| style_similarity | 30 |
| style_context | 힙·빈티지 취향 유저와 빈티지워시 데님 재킷이 잘 맞습니다. |
| **예상 Impulse** | **48~58** (D형 boost 있지만 20% 할인, U형인데 리뷰 95개는 M형 기준 아직 적음 → U boost 약) |
| **예상 Match** | **62~72** (스타일 맞음 + 1주 관심 + 검색 발견) |

**UM 긴장관계 체크:**
- U형 → 리뷰 95개: 검증은 됐지만 적은 편 → 약한 U 자극
- M형 → "한정수량/단독" 키워드에 오히려 끌림 (M형은 confidence 마케팅에 반응)
- 이미 결정했다고 했지만 Match가 중상위 → 또바: "마음은 정했는데 근거 확인해줄게" 방향

### Step 2: `01_build_scenarios.py` (4-2)

```
for each scenario in yaml:
    1. parse_sbti_flags(sbti)
    2. compute_impulse_score(...)   # 마케팅 트리거는 0/1 이진값 사용
    3. compute_match_score(...)     # style_similarity는 yaml에서 직접 입력
    4. build_system_prompt(...)
    5. build_context_sentences(...)
    6. 결과를 scenarios_built.json에 저장
```

출력 스키마:
```json
{
  "scenario_id": "S01",
  "label": "Q1-A / DUTE",
  "impulse_score": 79,
  "match_score": 60,
  "system_prompt": "...",

    "이거 사도 돼?",
    "이거 왜 이렇게 비싸?",
    "비슷한 거 이미 있는데..."
  ]
}
```

### Step 4: `03_evaluate.py` (4-4)
